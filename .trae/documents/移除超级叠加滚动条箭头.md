## 目标

- 去掉“超级叠加”Popover 内部两个滚动区域（已选列表、下拉列表）滚动条顶部/底部的上下箭头。

## 现状定位

- 已选列表滚动容器样式在 incomeChart 的 [index.less](file:///c:/Users/wusan/Desktop/myFEdemos/src/pages/Chart/incomeChart/index.less) 的 `.overlayDraftScroll`。
- 下拉列表滚动容器样式在同文件 `.overlaySelectPopup :global .rc-virtual-list-holder`。
- 两处目前已有 `::-webkit-scrollbar-button { display:none; ... }`，但在部分 Windows/浏览器/主题下仍可能出现，需要更细分的伪类覆盖。

## 修改方案

- 在上述两处补充更精确的 WebKit 伪元素选择器，覆盖常见的“单按钮/纵向/起止端”组合：
  - `::-webkit-scrollbar-button:single-button`
  - `::-webkit-scrollbar-button:vertical:decrement` / `:vertical:increment`
  - `::-webkit-scrollbar-button:vertical:start:decrement` / `:vertical:end:increment`
- 同时加上 `-webkit-appearance: none;` 与 `background: transparent;`，并保持 `width/height: 0`，确保箭头区域不占位。

## 验证方式

- 本地打开页面，点击“超级叠加”，分别：
  - 选择多个指数让已选列表出现滚动条；
  - 打开 Select 下拉并让下拉列表出现滚动条；
- 确认两处滚动条不再出现上下箭头，滚动与拖拽滚动条仍正常。
