# 三级菜单对应组件嵌套结构（Plan）

## 目标

实现“三级菜单 ↔ 内容区”严格的组件嵌套关系：

- 一级父组件：有自己的内容，并作为二级内容的容器
- 二级父组件：渲染在一级父组件内部；二级也有自己的内容，并作为三级内容的容器
- 三级子组件：渲染在二级父组件内部

对应到页面 DOM 结构期望为（概念示意）：

- Section(level0): Heading + 本级内容 + Section(level1...)
- Section(level1): Heading + 本级内容 + Section(level2...)
- Section(level2): Heading + 本级内容（叶子或末级）

## 现状确认（基于现有实现）

- 菜单树数据结构 `menuStructure` 已是三级树（key/title/children）。
- 滚动定位与高亮 `sections` 已拍平包含“所有层级节点”，因此父节点也能被定位与选中。
- 但内容区递归渲染 `renderNodes` 目前是二选一逻辑：有 children 就只渲染 children，没有则渲染叶子内容；导致父节点没有“本级内容”。
- `CatalogSection` 接收了 `title` 但未渲染标题；样式里已存在 `.nodeHeading` / `nodeLevel*`。
- `registry.tsx` 只为少量 key 提供“叶子内容”，并不支持父节点内容注册。

## 实施方案（最小改动、保持现有滚动同步）

### 1) 内容渲染策略：从“叶子才渲染内容”升级为“每个节点都渲染本级内容 + 子级递归”

在 `renderNodes` 里把原先：

- `hasChildren ? renderNodes(children) : renderLeafContent(node)` 改为：
- 永远先渲染本级内容 `renderNodeContent(node)`
- 如果有 children，再在本级内容后渲染子级 `renderNodes(children)`

这样即可保证一级/二级父节点也能拥有内容，同时三级仍在二级内部嵌套呈现。

### 2) 注册表升级：由 leafRegistry 扩展为 nodeRegistry（支持父节点内容）

将 `components/registry.tsx` 的“叶子注册表”概念升级为“节点注册表”：

- 保持现有 Overview / YieldDividend / VolHist / CompareStock 等内容组件可复用
- 新增父节点 key（如 `yield`、`yield-source` 等）可选映射到新建的内容组件（或占位组件）
- 未配置的节点统一走默认内容组件（现有 DefaultLeaf 可复用或改名为 DefaultNode）

备注：不强制所有父节点必须立刻有复杂内容；可先用默认占位，后续逐步补齐。

### 3) Section 组件完善：渲染标题 + 提供清晰的内容承载结构

改造 `components/CatalogSection.tsx`：

- 在 section 内部渲染标题节点（使用 `.nodeHeading` 样式）
- 将标题与 body 分开：Heading + Body
- Body 内直接渲染 children（children 将承载“本级内容 + 子级 Section 列表”）

### 4)（可选）父节点内容组件补齐

根据产品/演示需要，新增一些轻量“父节点内容”组件，例如：

- `Yield/index.ts` 或新增 `YieldSummary.tsx`：放收益率模块概览卡片/说明
- `YieldSourceSummary.tsx`：放收益来源概览优先做到结构正确，内容可逐步填充。

### 5)（可选）清理现有 UI 噪音

`LeafLayout.tsx` 若存在调试文案或不必要输出，可顺手清理（不影响结构，但能让页面更接近设计图）。

## 具体改动点清单（文件级）

- `src/pages/FrontendCard/CatalogScroll/index.tsx`
  - 调整 `renderNodes`：组合渲染“本级内容 + children”
  - 将 `renderLeafContent` 替换/升级为 `renderNodeContent`
- `src/pages/FrontendCard/CatalogScroll/components/registry.tsx`
  - leafRegistry → nodeRegistry（或保留命名但支持父节点 key）
  - `renderLeafContent` → `renderNodeContent`
- `src/pages/FrontendCard/CatalogScroll/components/CatalogSection.tsx`
  - 渲染 `title`（`.nodeHeading`）
  - 维持现有 level 样式逻辑
- （可选）`src/pages/FrontendCard/CatalogScroll/components/...`
  - 新增父节点内容组件或复用现有组件组合

## 验收要点（无需打包/代码检查）

- 内容区 DOM 层级满足：一级 section 内部包含二级 section，二级 section 内部包含三级 section。
- 一级/二级节点在内容区可见“本级内容”（至少是默认占位或轻量描述）。
- 点击目录任意层级节点可滚动到对应 section；滚动时目录选中项可随滚动更新（保持原逻辑）。
- 视觉上标题层级样式生效（`.nodeHeading` + level 样式）。
