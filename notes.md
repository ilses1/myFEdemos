候选清单（都支持手机+电脑）

- OneNote：最省心的“工作/生活日记本”，同步稳定，结构清楚（笔记本/分区/页面）
- Notion：适合“日志 + 复盘 + 清单/项目管理”，模板和数据库很强
- Obsidian（本地 Markdown）：适合“长期思考沉淀”，数据在你电脑里，同步靠云盘/Git
- Joplin（本地优先 + 可加密）：适合“隐私/加密同步”诉求更强的人我给你的最终推荐（不问你偏好，直接给最稳组合）

- 首选：OneNote（跨端最稳、上手最快、最不容易因为设置/同步折腾而中断记录）
- 备选 1：Notion（如果你希望把“每日记录”自然升级成“任务/项目管理 + 复盘系统”）
- 备选 2：Obsidian（如果你更在意“永远不被平台绑定、随时可迁移、离线也好用”）确保“数据不会丢失”的做法（不管选哪个都适用）

- 账号安全：开启两步验证（2FA），避免账号被盗导致数据风险
- 双保险备份：除了同步，再做“第二份备份”（另一云盘或移动硬盘）
- 定期导出：设一个频率（例如每月一次）
  - Notion：导出整个空间（Markdown/HTML/PDF 任选其一）
  - OneNote：至少确保 OneDrive 文件本身有备份/版本历史；关键内容可另存 PDF
  - Obsidian/Joplin：直接备份整个笔记文件夹/数据库目录即可
- 偶尔演练恢复：随机挑一次把导出文件打开/还原，确认备份真能用模板（可直接复制到任意笔记软件）

## Notion 最佳实践（从 0 到稳定可持续）

### 1) 先定“信息架构”，再谈模板

- 推荐骨架：PARA（Projects / Areas / Resources / Archive）
  - Projects：有明确目标与截止的事情（做完就结束）
  - Areas：需要长期维护的领域（健康、财务、学习、家庭）
  - Resources：资料/参考/可复用内容（文章、书摘、工具清单）
  - Archive：完成/不再活跃的内容
- 极简替代：只用 Projects + Resources + Archive 也可以跑起来

### 2) 先用“一个数据库多视图”，别一开始就到处建表

- Task 只建一个主库，再用视图分场景：Today / This Week / Waiting / Someday
- Project 只建一个主库，再用视图分状态：Active / Paused / Done
- 笔记/资料同理：一个库 + 标签/领域 + 多视图

### 3) 采集（Inbox）与整理（处理）分离，形成闭环

- Inbox：所有灵感/待办/资料先丢进来，不在采集时“想太多”
- 每天 10 分钟清空 Inbox：能 2 分钟内完成的直接做；否则改成下一步行动并安排时间/状态
- 每周回顾 30 分钟：看项目进度、清理拖延项、调整优先级

### 4) “结构化”只为检索与行动服务

- 标准化字段：状态 + 截止 + 所属项目 + 优先级（够用就停）
- 不要用过多标签：能用“所属项目/领域”解决的问题，别再加一层标签
- 不要把一切都做成数据库：纯文档（SOP/总结/文章）用页面就行

### 5) 逐步引入 Relation / Rollup（别一上来就复杂）

- 只有当你需要“跨表汇总/统计”时，再加关联与汇总
- 典型收益：
  - 项目页自动汇总任务完成情况
  - 领域页汇总正在进行的项目/习惯

## Notion 最小可用系统（建议直接照做）

### A. 3 个数据库就够用

1. Tasks（任务库）
2. Projects（项目库）
3. Notes / Resources（笔记/资料库）

### B. Tasks 字段建议

- Name（标题）
- Status（状态：Inbox / Next / Doing / Waiting / Done）
- Due（截止日期，可选）
- Priority（优先级：P1/P2/P3，可选）
- Project（关联 Projects）
- Area（领域，可选：健康/学习/工作/家庭）
- Tags（少量标签，可选）

### C. Projects 字段建议

- Name（项目名）
- Status（Active / Paused / Done）
- Goal（目标一句话）
- Start / End（开始/结束，可选）
- Area（领域）
- Tasks（关联 Tasks，用于项目下任务聚合）

### D. Home 页面（一个入口页面就够）

- 今日：Tasks 视图（过滤 Status in Next/Doing 且 Due = today 或 Priority = P1）
- 本周：Tasks 视图（Due 在未来 7 天）
- 等待：Tasks 视图（Status = Waiting）
- 收集箱：Tasks 视图（Status = Inbox）

## 7 天游学：每天 15–30 分钟，边学边搭

Day 1：块、页面、快捷输入、双链（[[页面]]）、基础排版 Day 2：数据库入门：表格/看板/日历/画廊，理解“一行就是一个页面” Day 3：过滤/分组/排序：学会“一个库多视图” Day 4：模板：任务模板、项目模板、会议纪要模板（默认字段 + 清单） Day 5：Relation + Rollup：让项目页自动汇总任务 Day 6：流程闭环：Inbox 清空 + 每周回顾清单固化 Day 7：精简与优化：字段瘦身、归档策略、共享与权限

## 维护规则（可复用）

- 命名规范：项目以动词开头（“搭建…/优化…/完成…”），任务以“下一步行动”描述
- 状态优先：同一时间只维护一个状态字段作为主流程
- 归档策略：Done 超过 30 天的任务自动/手动归档到 Archive 视图
- 每周回顾清单：
  - 清空 Inbox
  - 检查所有 Active Projects 是否有 Next Action
  - 标记/暂停不再推进的项目
  - 为本周选定 1–3 个最重要项目与对应下一步行动
